# 1C: выгрузка и загрузка конфигурации из файлов (CF / CFE)

Набор скриптов для **файловой информационной базы** и платформы **1С:Предприятие 8.3**: пакетный запуск конфигуратора (`1cv8.exe DESIGNER`) с типовыми ключами `/LoadConfigFromFiles`, `/UpdateDBCfg`, `/DumpCfg` (и при необходимости `/DumpDBCfg`).

Подходит для подключения в любой Git-репозиторий с выгрузкой конфигурации в XML (каталог с `Configuration.xml`).

### Обязательно от пользователя: путь к файловой базе

**Никто кроме вас не знает, где на диске лежит ваша файловая ИБ.** Вы **обязаны** один раз прописать путь в **`env.json`**: в поле **`default."--ibconnection"`** — строка вида `/F"C:\полный\путь\к\каталогуИБ"` (каталог той информационной базы, с которой будет работать конфигуратор). Без этого скрипты и ИИ-агент **не смогут** выбрать базу сами: они только читают `env.json` или параметр **`-InfoBasePath`** в командной строке.

**Для ИИ-агента:** если пользователь просит выполнить выгрузку (например: «сделай выгрузку CF и CFE»), открой этот README и следуй разделу **«Инструкция для ИИ-агента»** ниже. Все пути к XML и скриптам агент восстанавливает сам; **исключение** — реальный путь к ИБ: если в `env.json` его нет или стоит шаблон, агент **обязан** остановиться и **попросить пользователя явно указать путь к файловой базе** (или вставить готовый `env.json`).

### Минимальный набор для переноса в другой репозиторий

Скопируйте каталог **`Scripts`** (хотя бы `emit-vscode-tasks.ps1`), файлы **`Export-MainCf.ps1`**, **`Export-ExtensionCfe.ps1`**, **`cf.cmd`**, **`cfe.cmd`**, **`env.example.json`**, **`README.md`**, при необходимости **`.gitignore`**. Затем выполните `.\Scripts\emit-vscode-tasks.ps1` и создайте у себя **`env.json`** по образцу.

## Возможности

| Объект | Загрузить в ИБ из файлов | Выгрузить в архив | Загрузить + выгрузить одним запуском |
|--------|--------------------------|------------------|----------------------------------------|
| **Основная конфигурация** | `Export-MainCf.ps1 -Step LoadOnly` | `Export-MainCf.ps1 -Step DumpOnly` | `Export-MainCf.ps1 -Step All` (по умолчанию) |
| **Расширение** | `Export-ExtensionCfe.ps1 -Step LoadOnly` | `Export-ExtensionCfe.ps1 -Step DumpOnly` | `Export-ExtensionCfe.ps1 -Step All` (по умолчанию) |

- Основная конфигурация: выходной файл **`.cf`**, по умолчанию каталог **`cf\`** в корне репозитория.
- Расширение: выходной файл **`.cfe`**, по умолчанию каталог **`cfe\`** в корне репозитория. В командной строке используется имя расширения из `Configuration.xml` в каталоге файлов (`-Extension`).

## Требования

- Установленная платформа 1С 8.3 (скрипт ищет последнюю `1cv8.exe` в `Program Files\1cv8\…\bin\`).
- **Файловая ИБ** на вашем ПК: **путь к её каталогу вы задаёте сами** в `env.json` → `--ibconnection` (или `-InfoBasePath`). В этой ИБ уже должна открываться нужная конфигурация; для расширения — расширение **подключено** к основной с тем же внутренним именем, что в XML.
- Каталог с выгрузкой в файлы: для основной — **`base-conf`** (или свой путь через `-ConfigFilesPath`), для расширения — **`ext-ad`** (или `-ExtensionFilesPath`).

## Быстрый старт

1. Скопируйте **`env.example.json`** в **`env.json`**. **Обязательно замените** значение **`--ibconnection`** на **ваш реальный путь** к каталогу файловой ИБ на этом компьютере (см. блок выше «Обязательно от пользователя»). Дополнительно укажите учётные данные:

   - **`--ibconnection`** — только в формате `/F"C:\…\каталогИБ"` (скрипт берёт каталог после `/F`).
   - **`--db-user`**, **`--db-pwd`** — пользователь ИБ (пустой пароль допускается).

2. Полный цикл в один запуск:

   - основная конфигурация: **`cf.cmd`** или `.\Export-MainCf.ps1`;
   - расширение: **`cfe.cmd`** или `.\Export-ExtensionCfe.ps1`.

3. Проверка без запуска конфигуратора: добавьте **`-WhatIf`** (ничего не пишется на диск, кроме вывода в консоль).

## Инструкция для ИИ-агента

Ниже — единственный сценарий, который агент должен выполнить **самостоятельно**, когда пользователь просит выгрузить конфигурацию и расширение (формулировки вроде: «сделай выгрузку в CF и CFE», «прогони cf и cfe», «полный экспорт по скриптам»). Пользователь **не обязан** дублировать пути из README: агент берёт их из этого файла и из дерева репозитория.

### То, что пользователь обязан указать сам: путь к файловой базе

- **Путь к ИБ не выводится из репозитория и не угадывается.** В `env.json` в **`default."--ibconnection"`** должна быть строка подключения к **реально существующему** на этой машине каталогу файловой ИБ (`/F"C:\…\каталогИБ"`).
- Если **`env.json` нет** или в **`--ibconnection`** до сих пор шаблон из примера (`C:\Path\To\...` и т.п.) — **не запускай** `cf.cmd` / `cfe.cmd`. Сообщи пользователю: *нужно явно указать путь к файловой информационной базе* (вставить путь в `env.json` или дать строку `/F"…"` для подстановки). Альтернатива для продвинутых: передать **`-InfoBasePath`** при вызове скриптов, но по умолчанию ожидается именно `env.json`.

### Шаг 0. Среда

1. Убедись, что ОС — **Windows** (скрипты рассчитаны на `cmd` / PowerShell и `1cv8.exe` под Windows). Иначе сообщи, что автоматизация здесь не применима.
2. Определи **корень набора скриптов** — каталог, в котором одновременно существуют файлы **`Export-MainCf.ps1`** и **`Export-ExtensionCfe.ps1`**. В Cursor это обычно корень открытой рабочей области; если текущая папка глубже — поднимись по родителям, пока не найдёшь эти имена.
3. Все команды шагов 4–5 выполняй с **текущей директорией = этот корень** (`Set-Location` / `cd` в PowerShell).

### Шаг 1. Файл `env.json`

1. Если в корне есть **`env.json`** — используй его как есть (не выводи пароль в чат).
2. Если **`env.json` нет**, но есть **`env.example.json`** — скопируй: `Copy-Item -LiteralPath .\env.example.json -Destination .\env.json`.
3. После копирования открой `env.json` и проверь **`default."--ibconnection"`**: если путь выглядит как несуществующий шаблон или плейсхолдер — **не запускай** экспорт. Явно попроси **пользователя указать путь к файловой базе** (каталог ИБ на диске) в формате `/F"C:\…"` или прислать готовый фрагмент `env.json`.

При необходимости учётные данные — ключи **`--db-user`**, **`--db-pwd`** (как в примере); пустой пароль допустим.

### Шаг 2. Входные каталоги XML (проверка)

По умолчанию (см. таблицу путей ниже):

- **`base-conf\Configuration.xml`** — основная конфигурация;
- **`ext-ad\Configuration.xml`** — расширение.

Если какого-то файла нет — сообщи точный ожидаемый путь. Если в проекте XML лежит в другом месте — запускай с переопределением (подставь пути самостоятельно из структуры репозитория, если пользователь или проект их явно задают, иначе снова спроси пользователя):

```powershell
.\Export-MainCf.ps1 -ConfigFilesPath "…" -Step All
.\Export-ExtensionCfe.ps1 -ExtensionFilesPath "…" -Step All
```

### Шаг 3. Опционально — сухой прогон

Если пользователь просит только проверить аргументы без 1С на диске:

```powershell
.\Export-MainCf.ps1 -WhatIf
.\Export-ExtensionCfe.ps1 -WhatIf
```

### Шаг 4. Полная выгрузка (основная → CF, затем расширение → CFE)

Последовательно, **два отдельных** процесса конфигуратора (не параллельно):

```powershell
cd <корень из шага 0>
.\cf.cmd
# дождаться завершения; код выхода должен быть 0
.\cfe.cmd
```

Эквивалент без `cmd`:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\Export-MainCf.ps1 -Step All
powershell -NoProfile -ExecutionPolicy Bypass -File .\Export-ExtensionCfe.ps1 -Step All
```

### Шаг 5. Критерий успеха и диагностика

- **Успех:** в каталоге **`cf\`** появился новый файл вида `cf-export-*.cf`, в **`cfe\`** — `cfe-export-*.cfe`; оба разумного размера (не пустышки); команды завершились с кодом **0**.
- **Логи:** в консоли будет путь к объединённому логу в `%TEMP%`; копии — **`build-logs\last-main-cf-export.log`** и **`build-logs\last-cfe-export.log`**. При ошибке прочитай хвост этих файлов или фрагментов в каталоге `1c-export-*-fragments-*` под `%TEMP%`, путь печатается при запуске.
- **Типичные причины сбоя:** не закрыт конфигуратор по той же ИБ; неверный путь в `env.json`; несовпадение конфигурации/расширения с ИБ; нет установленной платформы 8.3 в стандартном каталоге `Program Files\1cv8`.

### Шаг 6. Что сообщить пользователю

Кратко: выполнены ли `cf` и `cfe`, полные пути к созданным `.cf` / `.cfe`, коды выхода, при ошибке — **файл лога** и 1–2 ключевые строки из хвоста лога.

## Пути по умолчанию (в корне репозитория)

| Назначение | Путь / файл |
|------------|-------------|
| Основная конфигурация (XML) | **`base-conf\`** (`Configuration.xml` внутри) |
| Расширение (XML) | **`ext-ad\`** |
| Выгрузка **.cf** | **`cf\cf-export-ГГГГММДД-ЧЧММСС.cf`** |
| Выгрузка **.cfe** | **`cfe\cfe-export-ГГГГММДД-ЧЧММСС.cfe`** |
| Локальные настройки ИБ (не коммитить с секретами) | **`env.json`** |
| Шаблон настроек для репозитория | **`env.example.json`** |
| Копия последнего объединённого лога (расширение) | **`build-logs\last-cfe-export.log`** |
| Копия последнего объединённого лога (основная) | **`build-logs\last-main-cf-export.log`** |

Параметры **`-OutCfPath`**, **`-OutCfePath`**, **`-ConfigFilesPath`**, **`-ExtensionFilesPath`**, **`-InfoBasePath`** переопределяют пути.

## Задачи VS Code / Cursor

Файл **`.vscode\tasks.json`** можно пересобрать командой:

```powershell
.\Scripts\emit-vscode-tasks.ps1
```

В списке задач появятся отдельные пункты для **LoadOnly**, **DumpOnly**, **All** и режимов **WhatIf** для основной конфигурации и расширения.

## Что коммитить в Git

**Рекомендуется включить в репозиторий**

- `Export-MainCf.ps1`, `Export-ExtensionCfe.ps1`
- `cf.cmd`, `cfe.cmd`
- `Scripts\emit-vscode-tasks.ps1`
- `env.example.json`
- `README.md`
- при необходимости — **`.vscode\tasks.json`** (или только генератор `emit-vscode-tasks.ps1`, а `tasks.json` пусть каждый генерирует локально)

**Обычно не коммитят (или коммитят только у себя)**

- **`env.json`** — содержит путь к ИБ и пароль; в общем репозитории держите только **`env.example.json`** без реальных данных. Если файл уже попал в индекс Git, уберите его из отслеживания: `git rm --cached env.json` (файл на диске останется).
- **`cf\*.cf`**, **`cfe\*.cfe`** — большие бинарные артефакты (при желании добавьте в `.gitignore`).
- **`build-logs\`** — отладочные копии логов.

**Каталоги выгрузки конфигурации в файлы (`base-conf`, `ext-ad`)**

- Для **типовой ERP** полный дерево XML может быть **очень большим**; решайте политику сами: полный коммит, Git LFS, субмодуль, отдельный артефактный репозиторий или игнор с выгрузкой «по месту» на машине разработчика.
- Для **минимального шаблона** этого репозитория достаточно скриптов + `env.example.json`; каталоги `base-conf` / `ext-ad` подключаются как у вас принято в проекте.

## Ограничения

- Запуск идёт через **конфигуратор**; параллельно с интерактивным конфигуратором по той же ИБ может быть конфликт блокировок.
- Режим **`-Engine Ibcmd`** есть только в **`Export-ExtensionCfe.ps1`**; для основной конфигурации в **`Export-MainCf.ps1`** используется только Designer.
- Выгруженный **`.cfe`** применим только к **совместимой** основной конфигурации и версии платформы.

## Полезные команды PowerShell

```powershell
# Основная: только загрузка из файлов в ИБ
.\Export-MainCf.ps1 -Step LoadOnly

# Основная: только выгрузка .cf (конфигурация в ИБ должна быть актуальна)
.\Export-MainCf.ps1 -Step DumpOnly

# Расширение: полный цикл с явным каталогом
.\Export-ExtensionCfe.ps1 -ExtensionFilesPath "D:\work\my-ext" -Step All
```

## Лицензия и состав конфигураций

Состав выгрузок 1С и лицензирование типовых конфигураций определяются вашими договорами с правообладателем; этот репозиторий содержит только **автоматизацию вызова** стандартных механизмов платформы.
